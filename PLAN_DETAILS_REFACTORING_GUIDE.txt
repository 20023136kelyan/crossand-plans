## 5. ENHANCED FEATURES

### USER CONTEXT & CONDITIONAL RENDERING REQUIREMENTS

#### USER ROLES & PERMISSIONS:

1.  **Plan Host** (`plan.hostId === currentUser.uid`):
    -   Full edit permissions
    -   Can manage participants and invitations
    -   Can delete plan
    -   Can view all participant responses
    -   Access to plan analytics/insights

2.  **Invited Participant** (`plan.invitedParticipantUserIds.includes(currentUser.uid)`):
    -   Can RSVP (yes/no/maybe)
    -   Can view other participants
    -   Can add comments and photos
    -   Can share plan with friends
    -   Cannot edit plan details

3.  **Confirmed Participant** (`plan.participantResponses[currentUser.uid] === 'going'`):
    -   All invited participant permissions
    -   Access to participant-only features (group chat, etc.)
    -   Can upload photo highlights
    -   Enhanced social features

4.  **Public Viewer** (not authenticated or not invited):
    -   Read-only access to public plan information
    -   Can copy plan as template
    -   Cannot RSVP or participate
    -   Limited social features

5.  **Authenticated Non-Participant** (logged in but not invited):
    -   Can request to join (if plan allows)
    -   Can copy plan as template
    -   Can follow plan for updates
    -   Cannot RSVP without invitation

#### CONDITIONAL UI ELEMENTS:

-   **RSVP Section**: Only visible to invited participants
-   **Participant Management**: Only visible to host
-   **Edit Controls**: Only visible to host
-   **Group Chat Access**: Only for confirmed participants
-   **Photo Upload**: Only for participants
-   **Copy Template Button**: Visible to all authenticated users
-   **Join Request**: For authenticated non-participants (if enabled)
